# UI/UX監査レポート — menskajiyaru.com

調査日: 2026-03-13
担当: UI/UXデザイナー

---

## 1. ファーストビュー（ヘッダー・トップ）

### 現状
- サイトロゴ: **テキストのみ「メンズ家事やる」**（画像ロゴなし）
- ヘッダー: ナビゲーションメニュー（11カテゴリ）＋テキストロゴ
- ファーストビューで見えるもの: ナビ・サイト名・記事2件の上半分

### 問題点
- **ロゴがテキストのみ**で、ブランドとしての視覚的認識が弱い。再訪問者が「このサイトに戻ってきた」と感じにくい。
- **サイトの「何を扱うサイトか」がすぐに伝わらない**。初訪問者はタイトルと小さなキャッチコピーのみで判断しなければならない。
- ナビゲーションに11カテゴリが並んでおり、**情報過多でどこから見ればいいか迷う**。

### 改善案
- シンプルなテキストロゴでも、フォント・色・サイズに統一感を持たせる（現状はデフォルトフォント）
- サブタイトル「家事やる夫の料理、家事、育児、仕事」をより読者への価値提案に変更（例:「40代パパが本音で語る仕事・家事・育児のリアル」）
- ナビゲーションカテゴリを4〜5つに整理

## 2. 記事一覧ページ（ホーム）

### 現状
- 2カラムグリッドレイアウト（記事 × 2 ＋ サイドバー）
- アイキャッチ画像: AI生成のアニメキャラクター（レッサーパンダ・キツネ）
- 記事カード: サムネイル・タイトル・著者・日付・カテゴリ・冒頭テキスト・READ MOREボタン

### 問題点
- **READ MOREボタンが装飾なしの細いテキストリンク**で、CTA（行動喚起）として弱い。
- **AI生成イラストが内容と乖離している**: 体験談・仕事・副業の内容に対してカラフルなアニメキャラクターは「ポップすぎる」印象で、40代男性読者層とのギャップがある。
- **記事カードに読了時間・人気度等の付加情報がない**: 読者が記事を選ぶ判断材料が少ない。

### 改善案（P2）
- READ MOREをボタン型CTA（背景色・角丸・テキスト「続きを読む →」）に変更
- アイキャッチ画像の方向性を検討（40代男性が共感できるリアル寄りのビジュアルか、一貫したキャラクターデザイン）
- 記事カードに「読了時間（約◯分）」を表示

## 3. 記事詳細ページ

### 現状
- 構成: ヘッダー → アイキャッチ → タイトル・メタ → 本文 → 前の記事リンク → コメント → フッター
- サイドバー: 検索・最新記事・転職ガイドリンク・カテゴリ・タグクラウド

### 問題点（重要）

| 問題 | 影響度 |
|------|--------|
| **目次（TOC）がない** | 長記事の読了率低下・離脱率上昇 |
| **関連記事セクションがない** | 読了後の離脱を防げない |
| **著者プロフィールボックスがない** | E-E-A-T・信頼性の欠如 |
| **SNSシェアボタンが記事内にない** | 拡散機会の損失 |
| **パンくずリストが表示されていない** | サイト構造の理解・SEO補助なし |
| **記事下の前後記事ナビが「Previous Post」のみ** | UIが英語・Nextなし |

- **「EDIT THIS」リンクが一般訪問者にも見える**（ログイン時のみだが、管理者がログインしたまま確認すると常に表示。スクリーンショット共有時に混乱を招く）
- **著者名が「MENSKAJIYARU」と全大文字の英字**：日本語サイトとして読者への親近感が薄い。

### 改善案
- **TOC（目次）プラグイン導入（P1）**: Easy Table of Contents等で自動生成
- **関連記事設置（P1）**: 記事末尾に3〜5件の関連記事カードを追加
- **著者プロフィールボックス追加（P1）**: 経歴・SNSリンク付きで信頼性UP
- **SNSシェアボタン設置（P1）**: 記事上下に配置（Share Buttons by AddThis等）
- **パンくずリスト表示化（P2）**: テーマ設定またはプラグインで有効化
- **前後記事ナビを日本語化（P3）**: 「← 前の記事」「次の記事 →」

## 4. フッター

### 現状
```
Designed using Dollah. Powered by WordPress.
```
**リンクが一切ない。**

### 問題点（P0: 法的リスク）
- **プライバシーポリシーページへのリンクがない**: Google Analytics等を利用している場合、プライバシーポリシーの掲載はGoogleのポリシー要件であり、日本の個人情報保護法でも求められる。
- **お問い合わせページへのリンクがない**: 読者・企業からの連絡手段がない。
- **プロフィール/Aboutページへのリンクがない**: 著者の信頼性確認ができない。
- **コピーライト表記がない**: 2026年表記も存在しない。

### 改善案（P0）
フッターに最低限以下を追加:
- プライバシーポリシー（新規ページ作成）
- お問い合わせ（Contact Form 7等で作成）
- プロフィール
- コピーライト「© 2026 メンズ家事やる」

## 5. モバイル表示

### 現状
- ハンバーガーメニューあり（「≡」アイコン）
- 全体的なレスポンシブ対応はできている
- WordPress管理バーがモバイル上部に表示（管理者ログイン時）

### 問題点
- **管理バーがモバイルの表示領域を占有**: 一般読者には関係ないがスクリーンショット共有時に不格好。→ `show_admin_bar(false)` で非表示可
- **記事メタ情報（著者/日付/タグ/EDIT THIS）が1行に並んでモバイルで潰れる**: 視認性が低い。

### 改善案（P3）
- モバイルでの記事メタを「日付・カテゴリ」のみに簡略化
- 管理バーをフロントエンドで非表示（管理画面では引き続き表示）
