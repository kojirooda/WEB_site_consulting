## 1. メタ情報

### 現状
| 項目 | 現状値 | 基準 | 評価 |
|------|--------|------|------|
| ホームtitle | "メンズ家事やる - 家事やる夫の料理、家事、育児、仕事" | 30〜60字 | ✅ |
| ホームmeta description | "家事やる夫の料理、家事、育児、仕事"（17文字） | 80〜160字 | ❌ |
| 記事meta description | 300文字（AGA記事） | 80〜160字 | ❌ |
| H1（ホーム） | "メンズ家事やる"（サイト名のみ） | キーワードを含む | ❌ |
| H1（記事） | 記事タイトルと一致 | 1ページ1つ | ✅ |
| canonical | 設定済み | — | ✅ |

### 問題点
- **ホームmeta descriptionが17文字**と極端に短く、検索結果でGoogleが自動生成した説明文が使われてしまう。CTR（クリック率）に直接影響する。
- **記事meta descriptionが300文字**と長すぎる。Googleは155文字前後で切り捨てる。
- **ホームH1がサイト名のみ**。「40代 男性 家事」「パパ 副業」等の検索意図キーワードが含まれない。

### 改善案
- ホームmeta description: 「40代経営者・3児の父が実践する家事・育児・仕事・副業のリアル。男性目線の生活情報メディア。」（60文字）
- 記事meta descriptionを各記事で120字以内に再設定（Yoast SEOのフォーカスキーフレーズ設定で管理）
- ホームH1を非表示にしてdescription的なコピーに変更するか、テーマのキャッチフレーズをキーワード含有のものに変更

## 2. URL構造

### 現状
```
❌ 現状: /2026/03/11/aga%E6%B2%BB%E7%99%82%E3%82%92%E3%81%AF%E3%81%98%E3%82%81%E3%81%A6%E7%B4%845%E5%B9%B4/
✅ 理想: /aga-chiryo-5nen/
```

### 問題点
1. **URLに日付が含まれる**（`/2026/03/11/`）：記事が古く見え、リライト時にURLが変わらないため混乱が生じる。Googleも日付なしURLを推奨傾向。
2. **URLスラッグが日本語**（URLエンコードされた文字列）：シェア時に文字化け・URL肥大化が起きる。SNSでの見た目が悪く、OGP共有時の問題にもなる。
3. 現在の設定はWordPressの「日付と投稿名」パーマリンク構造。

### 改善案（P1）
- Yoast SEOの投稿スラッグをローマ字またはハイフン区切り英語に変更（例: `aga-chiryo-5nen`）
- WordPressパーマリンク設定を「投稿名のみ」に変更: `/aga-chiryo-5nen/`
- **注意**: 既存URLが変わるため301リダイレクト設定が必須

## 3. 構造化データ（Schema.org）

### 現状（記事ページ）
```
✅ Article, WebPage, ImageObject, BreadcrumbList, WebSite, Person
```
Yoast SEOが自動生成している。基本的なスキーマは存在する。

### 問題点
- **FAQスキーマなし**: Q&A形式の記事（AGA記事）にFAQスキーマを追加するとリッチリザルト表示の可能性がある。
- **ReviewスキーマなしProduct系記事**: クリニックや薬品の体験記事に Rating/Review スキーマを追加できる。
- **BreadcrumbListはスキーマ上存在するが画面に表示されていない**（後述UI/UX参照）。

### 改善案（P2）
- Q&A形式の記事にFAQ schemaをYoast SEOのブロックエディタから追加
- 転職・キャリア系記事にHowTo schemaを検討

## 4. 内部リンク・サイト構造

### 現状
- 記事ページの内部リンク数: 129（サイドバー多数のリンクを含む）
- 記事本文内の関連記事リンク: **なし**（記事末尾の関連記事セクションが未設置）
- カテゴリ数: 11（多数のカテゴリに記事数1〜6）

### 問題点
- **記事本文内の内部リンクがほぼない**: 読者を他記事へ誘導できず、サイト回遊率が低下。Googleのクローラビリティにも影響。
- **カテゴリが11個と多く分散しすぎ**: 「政治経済」「ガジェット」等は記事が少なく薄いカテゴリページになっている。
- **関連記事ブロックが未設置**: 記事末尾に関連記事がなく、離脱率が高い原因になっている。

### 改善案（P2）
- 各記事本文内に最低2〜3本の内部リンクを追加
- カテゴリを「仕事・転職」「家事・育児」「体験談・個人」「特集」の4つに統合を検討
- 関連記事プラグイン（Contextual Related Posts等）の導入

## 5. OGP・SNSメタ

### 現状
| 項目 | 状態 |
|------|------|
| og:title | ✅ 設定済み |
| og:description | ❌ "Read More →" が混入（約80字の途中で切れている） |
| og:image | ✅ 設定済み（/wp/wp-content/uploads/...） |
| og:type | ✅ article |
| twitter:card | ✅ summary_large_image |
| twitter:site | ❌ 未設定（none） |
| twitter:creator | ❌ 未設定 |

### 問題点
- **og:descriptionに"Read More →"が混入**: Twitterなどでシェアされた際に「Read More →」という文字が表示されてしまう。Yoast SEOの設定かテーマのexcerpt生成の問題。
- **twitter:siteが未設定**: XのアカウントIDをYoast SEO設定に入力する必要がある。

### 改善案（P1）
- Yoast SEO → ソーシャル設定でXアカウントを設定
- og:description/twitter:descriptionをYoast SEOの「ソーシャルの説明」フィールドで明示指定
