## 1. ページ読み込み速度

### 計測値（記事ページ）
| 指標 | 計測値 | 目標値 | 評価 |
|------|--------|--------|------|
| DOMContentLoaded | **2,498ms** | < 1,000ms | ❌ |
| Load Complete | **2,850ms** | < 2,000ms | ❌ |
| 総リソース数 | **84件** | < 40件 | ❌ |
| CSS ファイル数 | **16ファイル** | < 3ファイル | ❌ |
| JS ファイル数 | **43ファイル** | < 5ファイル | ❌ |
| 転送サイズ | ~50KB（キャッシュ後） | — | △ |

### 問題点
- **Autoptimizeプラグインが入っているにも関わらず16 CSS + 43 JSが読み込まれている**: Autoptimizeの設定が不十分か、一部プラグインがAutoptimizeの対象外になっている可能性。
- DOMContentLoadedが2.5秒はスマートフォン回線（4G）では4〜5秒以上になる可能性がある。
- Core Web Vitalsの LCP（Largest Contentful Paint）が悪化している可能性が高い。

### 改善案（P1）
- Autoptimizeの設定を見直し: CSS・JS・HTMLの最小化と結合を有効化
- 不要プラグインの無効化・削除
- より強力なキャッシュプラグイン（W3 Total Cache / WP Rocket）の検討

## 2. 画像最適化

### 現状
| 指標 | 計測値 | 評価 |
|------|--------|------|
| ページ内画像数 | 18枚 | — |
| lazy-load対応 | **0枚**（loading="lazy"なし） | ❌ |
| alt属性なし画像 | **4枚** | ❌ |
| 画像フォーマット | JPEG（WebP未使用） | ❌ |
| OGP画像パス | `/wp/wp-content/uploads/...` | — |

### 問題点
- **lazy-loadingが一切設定されていない**: スクロールしないと表示されない画像（記事中・サイドバー）も最初から全て読み込んでしまい、初期表示が重い。
- **WebP形式が使われていない**: JPEGと比較してWebPは25〜35%のファイルサイズ削減が期待できる。
- **alt属性のない画像が4枚**: アクセシビリティ問題かつ画像SEOに悪影響。

### 改善案（P1）
- WordPress 5.5以降では`loading="lazy"`がデフォルトで付与されるが、テーマ・プラグインが上書きしている可能性。設定確認が必要。
- ShortPixel / Imagify / WebP ExpressプラグインでWebP自動変換を導入
- alt属性の設定漏れを一括確認・修正

## 3. CSSとJavaScript

### 現状
- 16個のCSSファイルが読み込まれている: テーマCSS、プラグインCSS（SiteKit、Autoptimize、ブロックエディタ等）
- 43個のJSファイル: jQuery、WordPressコア、Yoast、SiteKit、AddThis（？）、ブロックエディタライブラリ等
- フォント読み込み: Google Fontsまたはシステムフォント（要確認）

### 問題点
- **JSファイル43本はWordPressの典型的な問題**: 各プラグインが独自のJSを注入している。
- 使用されていないCSSルール（未使用のブロックエディタスタイル等）が含まれている可能性。

### 改善案（P1）
- `WP Asset CleanUp` プラグインで各ページに不要なCSS/JSを個別に無効化
- Autoptimize設定で「CSSを最適化」「JSを最適化」「HTMLを最適化」をすべて有効化
- Critical CSS（Above-the-foldのCSS）のインライン化を検討

## 4. ホスティング・インフラパフォーマンス

### 現状
- **WordPress設置パス**: `/wp/` サブディレクトリ（ルートからリダイレクト）
- HTTPS: 有効 ✅
- CDN: 不明（要確認）

### 問題点
- CDNが未設定の場合、画像・静的ファイルの配信が遅い。
- WordPressが `/wp/` サブディレクトリ設置のため、リソースパスが長くなりデバッグが複雑。

### 改善案（P2）
- Cloudflare無料プランの導入でCDN・DDoS対策・キャッシュを一括改善
- またはCloudflare APOで WordPressのHTML自体をエッジキャッシュ
