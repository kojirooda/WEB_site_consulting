# WEB_site_consulting — プロジェクト定義

## プロジェクト概要
クライアントのウェブサイトを多角的に調査・分析し、改善提案と実装を行うコンサルティングプロジェクト。

## チーム編成

| ロール | 担当領域 | 主な成果物 |
|--------|----------|-----------|
| **PM / ディレクター** | 全体進行・優先度管理・クライアント調整 | プロジェクト計画、改善提案書 |
| **UI/UXデザイナー** | 導線設計・視覚的一貫性・モバイル対応・CTA設計 | UX監査レポート、改善デザイン案 |
| **フロントエンドエンジニア** | HTML/CSS品質・パフォーマンス・アクセシビリティ・Core Web Vitals | コード改善実装 |
| **SEOスペシャリスト** | キーワード戦略・構造化データ・内部リンク・メタ情報 | SEO監査レポート |
| **コンテンツストラテジスト** | 記事品質・E-E-A-T・読了率・CTA文章 | コンテンツ改善提案 |
| **SNS/集客マーケター** | OGP・SNS連携・流入経路設計・CV導線 | 集客改善提案 |
| **インフラエンジニア** | デプロイパイプライン・ホスティング・セキュリティ・URL設計 | インフラ改善提案 |

## 作業フロー

```
1. AUDIT（調査）   → 各担当が独立して監査レポートを作成
2. REPORT（提案）  → 改善提案書（proposals/）にまとめて確認依頼
3. IMPLEMENT（実装） → 承認後、対象リポジトリに実装
4. VERIFY（検証）  → 実装後の動作確認・効果測定
```

## ディレクトリ構成

```
WEB_site_consulting/
├── CLAUDE.md               # このファイル（プロジェクト定義）
├── team.md                 # チーム詳細定義・役割分担
├── audit/                  # 各担当の監査レポート
│   ├── ux.md
│   ├── seo.md
│   ├── content.md
│   ├── sns.md
│   ├── performance.md
│   └── infra.md
└── proposals/              # 改善提案書（クライアント確認用）
    └── improvement-proposal.md
```

## 優先度基準

| 優先度 | 基準 |
|--------|------|
| P0 — 即対応 | SEOクロール不可・サイト表示不能・重大セキュリティ問題 |
| P1 — 高 | CV率に直結・離脱率の高い導線問題・構造化データ欠損 |
| P2 — 中 | UX改善・コンテンツ品質向上・SNS拡散性 |
| P3 — 低 | デザイン微調整・将来的なスケール対応 |

## ルール
- 改善提案は必ず `proposals/improvement-proposal.md` にまとめ、クライアント確認後に実装する
- 対象リポジトリのファイルは提案承認前に変更しない
- 各監査レポートには「現状」「問題点」「改善案」の3点セットで記載する
- コード変更は最小限・的を絞った修正を原則とする
